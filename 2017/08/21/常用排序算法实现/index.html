常用排序算法实现

发表于 2017-08-21 | 分类于 算法 |

1.直接插入排序

算法思想

将待排序的记录插入到已排序的子文件中去，使得插入之后得到的子文件仍然是有序子文件。
插入一个记录前，首先要对有序子文件进行查找，以确定这个记录的插入位置。

C语言实现

时间复杂度为O(n2)

2.交换排序——冒泡排序

算法思想

在要排序的一组数中，对当前还未排好序的范围内的全部数，自上而下对相邻的两个数依次进行比较和调整，让较大的数往下沉，较小数的往上冒。即：每当两相邻的数比较后发现它们的排序与排序要求相反时，就将它们互换。
对n个数来说，常规的冒泡排序要进行n-1趟，即使后面几趟都没有进行交换。因此可以对其进行改进，添加一个标志位，标志该趟排序是否交换了数据，若没有交换，表示这组数已经有序，不用再进行后面的几趟排序。

C语言实现

时间复杂度为O(n2)

3.交换排序——快速排序

算法思想

1.在a[1..n]中，确定一个基准元素a[i]。
2.经过一趟比较和移动，将a[i]放到”中间”某个位置上，使得a[i]左边所有记录的元素值小于等于a[i]，a[i]右边所有记录的元素值大于等a[i]。以a[i]为界,将文件划分为左、右两个子文件。
3.用同样的方法分别对这两个子文件进行划分, 得到4个更小的子文件。继续进行下去,使得每个子文件只有一个记录为止,便得到原文件的有序文件。

C语言实现

时间复杂度为O(nlogn)，但在最坏情况下（基本有序时），其时间复杂度为O(n2)

4.选择排序——简单选择排序

算法思想

在要排序的一组数中，选出最小（或者最大）的一个数与第1个位置的数交换；然后在剩下的数当中再找最小（或者最大）的与第2个位置的数交换，依次类推，直到第n-1个元素（倒数第二个数）和第n个元素（最后一个数）比较为止。

C语言实现

时间复杂度为O(n2)

5.选择排序——堆排序

堆排序是一种树形选择排序，是对简单选择排序的有效改进。

算法思想

堆的定义：n个元素的序列{k1,k2,…,kn} 当且仅当满足下关系时，称之为堆。
1.小顶堆：ki ≤ k2i 且 ki ≤ k2i+1
2.大顶堆：ki ≥ k2i 且 ki ≥ k2i+1
(i=1,2,…,n/2)
堆排序过程：初始时把要排序的n个数的序列看作是一棵顺序存储的二叉树（一维数组存储二叉树），调整它们的存储序，使之成为一个堆。堆的根节点的数最小（或者最大），将堆顶元素输出，就得到n 个元素中最小(或最大)的元素。然后对前面(n-1)个元素重新调整使之成为堆，输出堆顶元素，得到n 个元素中次小(或次大)的元素。依此类推，最后得到有n个节点的有序序列。
堆排序需解决两个问题：
1.如何将n个待排序的数处理成堆（大顶堆或小顶堆）。
2.如何在堆顶元素被替换后，调整剩余元素成为一个新的堆。

C语言实现

时间复杂度为O(nlogn)

6.2-路归并排序

算法思想

把2个有序子文件合并在一起，形成一个新的有序文件。同时归并2个有序子文件的排序过程称为2-路归并排序。

C语言实现

时间复杂度为O(nlogn)
